# Exhibit 05 — Semantic Mapping

## Simple layer

Big technical work does not live neatly in one folder.

It spreads across notes, downloads, drafts, zip files, duplicate names, logs, and half-finished versions.

A semantic map turns that mess into something humans and AI can navigate.

## Technical layer

This line separates:

- raw inventory
- semantic snapshot
- classified layer
- governance state
- dedupe decision
- terminal / unresolved / handoff-required state

## Boundary

Classification is not the same as deep understanding. Coverage is not the same as proof. A map is not the archive itself.
